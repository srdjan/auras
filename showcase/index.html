These defaults are intentionally restrained: readable typography, sensible spacing, semantic lists, links, code, tables, and surfaces that do not force a visual language.

Long-form primitive

Readable editorial rhythm without classes

Use data-ui="prose" when content needs a tighter reading measure, calmer spacing, and stronger heading rhythm.

This keeps application chrome separate from article copy while preserving Aura's semantic, attribute-first model.

  • Headings and lead copy gain clearer spacing.
  • Lists become easier to scan in long-form content.
  • Blockquotes and figure captions inherit the active tokens.

Scoped prose is one of the most transferable ideas from the external UI repo because it improves readability without forcing a new class-heavy component model.

Pilot: splitter

aura-splitter keeps both panes and the separator in authored light DOM, then adds keyboard resizing, drag resize, and percent-based layout state.

Current split: 42% primary pane

Focus the divider and use Left/Right, or drag it with the pointer, to resize the two panes.

Primary pane

This side is useful for navigation, a source list, or a compact inspector. The component only owns resize state.

  • Pointer drag updates the split.
  • Keyboard resize uses the configured step.
  • Home and End jump to min and max.

Secondary pane

Use this side for detail content, preview content, or a larger editing surface. Layout styling stays in the Composites layer.

The current value is exposed through the host value attribute and the aura-change event.

Auto activation: elements

Elements

This stays the base Aura file: semantic defaults, themes, layout primitives, forms, utilities, and neutral surfaces.

  • No JavaScript required.
  • Brand packs override tokens instead of selectors.
  • Everything remains easy to inspect and override.

Composites

Composites add stronger app patterns without taking ownership of behavior. This master-detail shell is one of those patterns.

  • CSS-only layout and responsive structure.
  • Stable data-part hooks for styling.
  • Still optional and still framework-agnostic.

Components

Components stay small and behavioral. The custom element here only manages selection, roving focus, keyboard movement, and panel visibility.

  • Light DOM only.
  • No shadow root and no hidden template contract.
  • Author-written HTML remains the source of truth.
